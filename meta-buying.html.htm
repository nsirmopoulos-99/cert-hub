Meta Business Manager (Unit 2)

Structure & Hierarchy

  • Business Portfolio The umbrella entity holding all assets. Requires a Business ID.
  • Assets Pages, Ad Accounts, Pixels, Catalogs, Apps. Grouped into Asset Groups for easy permission management.
  • People vs Partners People are individuals (employees). Partners are other businesses (agencies/clients) you share assets with.

Access Levels

Full Control (Admin): Manage settings, people, payment methods, and all assets.
Partial Access (Employee): Work only on assigned tasks/assets.
Advanced Options: Finance Analyst (View bills) vs Finance Editor (Edit credit cards).

Buying Types (Unit 4)

⚖️ Auction (Default)

  • Pricing: Fluctuates (CPM not fixed).
  • Goal: Max efficiency/value.
  • Flexibility: Can optimize for any objective.

📅 Reservation (Reach & Frequency)

  • Pricing: Fixed CPM. Guaranteed Delivery.
  • Goal: Predictability (Brand launches).
  • Limit: Only Awareness/Engagement objectives. Min 200k reach.

Managing Audiences (Unit 6)

1. Core / Detailed Targeting

Location, Age, Gender, Interests, Behaviors.
Advantage+ Audience: Allows Meta to go broader than your inputs if it finds cheaper conversions.

2. Custom Audiences (Warm)
Sources: Website (Pixel), App (SDK), Customer List (Email/Phone), Engagement (Video views, IG interaction).
3. Lookalike Audiences (Scaling)

Finds new people similar to a source. Requires 100+ people in seed (1k+ recommended).
1% = High Similarity (Narrow), 10% = High Reach (Broad).

Special Ad Categories (HEC): Housing, Employment, Credit. Restricted targeting (No Age, Gender, Zip Code targeting).

Signals & Data Sources (Unit 5)

Data sources connect customer actions (signals) to Meta for optimization, measurement, and targeting. Exam Tip: Understand when to use which tool.

🌐 Meta Pixel (Browser-Side)

A piece of JavaScript code on your website that tracks visitor activity. Vulnerable to ad blockers and cookie restrictions.

Standard Events

Pre-defined by Meta. Recognized automatically for optimization. Examples: Purchase, AddToCart, Lead, CompleteRegistration.

Custom Events

Actions you define. Must be mapped to a Custom Conversion to be used for optimization or reporting. Example: Scrolled 50% of page.

Tools: Use Meta Pixel Helper (Chrome Extension) to verify it's firing correctly. Use Advanced Matching to hash customer data (email/phone) for better attribution.

🔌 Conversions API (Server-Side)

Sends web, app, and offline events directly from your server to Meta. More reliable than Pixel as it bypasses browser issues.

  • Reliability: Not affected by browser crashes or ad blockers.
  • Full Funnel: Captures lower-funnel events (Qualified Lead) that happen off-site.
  • Best Practice: Use BOTH Pixel + CAPI for the same events.
⚠️ Deduplication: If using both Pixel and CAPI, you must include an Event ID and Event Name so Meta knows to count the event only once.

📱 Facebook SDK (App Tracking)

The equivalent of the Pixel for Mobile Apps. Required for App Promotion campaigns (Installs/Events).

App Events

1. Automatic: Installs, App Launches, Purchases (if integrated).
2. Standard: Achieved Level, Rated App.
3. Custom: Specific niche actions.

Deep Linking

Deep Link: Sends existing users directly to specific content (e.g., Red Shoes page).
Deferred Deep Link: Sends new users to App Store → Install → Then opens specific content.

Aggregated Event Measurement (AEM)

Protocol for iOS 14.5+ users who opt-out of tracking.
• Limits domains to 8 Prioritized Conversion Events.
• Reporting may be delayed (up to 3 days) and modeled.

Offline Conversions

Connects physical world data (CRM, POS systems) to Meta.
• Upload CSV files or use API.
• Allows you to attribute an in-store purchase to a digital ad view.

Creative Strategy (Unit 7)

Placements

Advantage+ Placements (Recommended): Meta's AI places ads across FB, IG, Messenger, and Audience Network based on lowest cost per result.
Manual Placements: Only use for brand safety or specific creative limits.

Key Ad Formats

Carousel: Up to 10 images/videos. Good for storytelling or product catalog.
Collection: Cover media + product thumbnails below. Opens into Instant Experience.
Instant Experience: Full-screen mobile landing page. Loads instantly (Native).
Creative Fatigue: Performance drops due to overexposure.
Fixes: Refresh creative, Expand audience, Use Advantage+ Creative (AI optimization).

Advertising Standards (Unit 8)

Prohibited (Not Allowed)

  • Personal Attributes: "Do you have diabetes?" (Implies knowledge of user).
  • Sensational Content: Shocking, gore, excessive violence.
  • Misleading Claims: "Lose 10lbs in 2 days!" (Unrealistic results).

Restricted (Requires Permission)

  • Alcohol (Must follow local age laws).
  • Online Dating.
  • Online Gambling.
  • Financial Services / Crypto.
Ad Review Process: Most ads are reviewed by automated machines. If rejected, you can edit and re-submit, or Request a Review (Appeal) if you believe it was a mistake.

Attribution & Reporting

Attribution Setting

Determines how credit is given to ads.
Default: 7-day click, 1-day view.
(Conversion counts if user clicked in last 7 days OR viewed in last 1 day).

Metrics

  • CPM: Cost per 1,000 impressions (Cost / Impressions * 1000).
  • ROAS: Revenue / Ad Spend.
  • CTR: Clicks / Impressions.
